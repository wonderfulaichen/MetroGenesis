## 二、Phase 1 测试策略

### 测试顺序（从最稳定到最不稳定）

推荐按此顺序逐步验证，任一阻断级问题解决后再继续：

```
Step 1: 基础启动测试（P0）
  → 加载 mod、查看 mod list、creative tab、无恐慌崩溃

Step 2: 命令测试（P1）
  → /cvalue list、/cvalue check 输出

Step 3: BlockUI 框架测试（P1）
  → BlockStateTestGui、ScrollingListsGui

Step 4: RTS 相机测试（P0）
  → 切换视角、移动、无崩溃

Step 5: 市长管理书测试（P0）
  → 获取物品、右键打开、浏览页面、3D 鸟瞰

Step 6: 蓝图系统测试（P0）
  → BuildToolUI 打开、选区框选、扫描保存、旋转/镜像

Step 7: 施工系统测试（P0）
  → 放置 ConstructionMarkerBlock、选择建筑、确认放置

Step 8: 经济系统测试（P1）
  → C-Value 初始化、市场数据输出

Step 9: 道路系统测试（P1）
  → 区块加载触发 RoadPipelineController、道路生成

Step 10: 市民系统测试（🔴高风险）（P0）
  → 招募市民、实体生成、AI 寻路、市民交互界面

Step 11: 市政厅测试（🔴高风险）（P0）
  → 放置 TownHallBlock、打开 TownHallScreen、完整功能
```

### 优先级分布

两套冒烟框架合并去重后：

| 优先级 | 数量 | 覆盖核心 |
|--------|------|----------|
| 🔴 P0（阻断级） | ~22 项 | 所有核心操作路径 |
| 🟡 P1（困惑级） | ~28 项 | 边缘情况、空状态、UX 反馈 |
| 🔵 P2（建议级） | ~5 项 | 体验优化、配色统一、动画平滑度 |
| 📐 需甲方实测 | ~6 项 | 联机同步、大面积性能、FOV 平滑度 |

## 三、已知问题与运行时风险

### 🚨 必须在测试前修复或确认的问题

#### 1. 中文编码损坏（阻断级，涉及4个文件）

| 文件 | 行 | 问题 |
|------|----|------|
| `CitizenEvents.java` | 20 | 日志中文字符编码损坏 |
| `CitizenManager.java` | 242 | 性别符号 "♀♂" 编码错误 |
| `ConstructionSite.java` | 94-122 | 多行中文日志 + §6 控制码 |
| `ConstructionSite.java` | 105 | 彩色消息文字编码解析错误 |

**建议**：将这些转换为英文日志，或确保被排除（`citizen/CitizenEvents.java` 已在排除列表中）。

#### 2. Java 语法错误（运行时可能触发）

| 文件 | 问题 |
|------|------|
| `FarmerWorkGoal.java:146-178` | `instanceof` 模式匹配变量用法错误 |
| `DailyScheduler.java:47` | `level.getDayTime() / 24000;` 缺少赋值 |
| `ConstructionSite.java:145` | 缺少 `;` |

#### 3. 运行时崩溃高风险模块

- **市民实体 (MetroGenesisCitizen)** — 可能引用被排除的 MineColonies 依赖 → NPE / LinkageError
- **GUI 中输入事件** — `TownHallScreen`, `CitizenInteractionScreen` 中的断点/断掉的逻辑
- **ConstructionSite.placeBuilding()** — 中文字符串拼接 § 控制码可能抛异常
- **任何触发 `job/` 或 `workorder/` 包的路径** — 整个包被排除

### ⚠️ 环境坑位

| 坑位 | 说明 | 建议 |
|------|------|------|
| Gradle 8.8 + JDK 21 | deprecation 警告，可能在 AT 处理时出问题 | 降级到 JDK 17 或升级 Gradle |
| 项目路径含中文字符 | 某些工具解析异常 | 保持当前路径不变量 |
| AT (accesstransformer) 未经验证 | 可能引用了已排除或结构不同的类 | 运行时验证 AT 生效 |
| 嵌入 MineColonies 代码零编译验证 | `com/ldtteam/**` 和 `com/minecolonies/**` 完全被排除编译 | 启用前先编译验证 |

## 四、需在测试中确认的技术问题

取自品鉴行-2 框架的前置确认清单，建议实际跑测试前先确认技术实现方式：

1. **蓝图存储格式**：体积预估（64×64×256=104万方块场景下的序列化效率）
2. **市民实体类型**：是否为 `Mob` 子类？是否设置了伤害豁免？
3. **区块加载保护**：市民寻路到未加载区块时的处理
4. **GUI 配色一致性**：当前存在3套不一致配色（BuildTool=蓝紫深色, MayorBook=磨砂玻璃, ConstructionMarker=Forge默认）
5. **权限模型**：是否需要 OP？是否支持普通玩家？
6. **区域规划标记方案**：粒子/实体/渲染层？
7. **按钮防连点**：确认按钮应有冷却（建议 500ms）
