## 风格包 Bug：Windows 路径崩溃

### 根因
`ClientStructurePackLoader.java:88` 和 `ServerStructurePackLoader.java:89`：

```java
modPath.toString().split("/")[1]  // Linux 可跑，Windows 必崩
```

Windows 下路径分隔符是 `\`，`split("/")` 返回单元素数组，`[1]` 抛 `ArrayIndexOutOfBoundsException`。该异常不是 `IOException`，外层 catch 不捕获 → 整个风格包加载静默崩溃 → 23 个包全未注册。

### 修复
```java
modPath.getFileName().toString()  // 跨平台安全
```

## 建筑系统设计讨论

### 核心矛盾
模拟城市的核心交互是"划区 → 自动生长"，但 MC 是方块世界，无法实现 SimCity 式的无缝街区。

### 解决方向：拼图式对齐建筑
- 在现有图鉴体系中，为每个风格包增加 `seamless/` 子目录
- 目录下的建筑特意设计成有平墙侧面的"拼图块"
- `StandardBlueprintData` 扩展 `SeamlessType` 枚举：
  - NONE / LEFT_WALL / RIGHT_WALL / BOTH_WALLS / INNER_CORNER / OUTER_CORNER
- 放置时自动检测隔壁是否为同类建筑 → 侧面吸附对齐
- 自动放置时：先用 seamless 建筑排满临街面，再用独立建筑填充内部

### 自动放置流程
```
划住宅区 → 系统扫描可建地块 →
→ 从 seamless/ 选拼图建筑沿街道排开（自动对齐侧面）→ 形成连续街面
→ 用 buildings/ 独立建筑填充剩余空地
→ 转角自动选 CORNER 建筑
```

### 不对的地方
- 不要追求"建筑之间无缝"（MC 方块世界不可能）
- 用"密度 + 风格统一 + 道路连接"营造城市感
- 玩家可自定义/接管放置（手拖蓝图到指定位置）
