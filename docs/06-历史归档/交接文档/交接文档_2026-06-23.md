# Handoff: MetroGenesis 图鉴系统全面完善 + 需求面板 + 交互修复

## Current state

Phase 0 地基系统大部分完成，遗留项集中于图鉴 UI 增强和撤销深度优化。

### 本会话完成的核心工作

**1. 图鉴放置流程修复**（`MayorBookScreen` + `BlueprintPlacementMessage`）
- 修复 mouseClicked 缺少 placement 处理的交互断裂
- 修复服务端用 BlueprintLibraryData 查找、客户端用 StructurePacks 的数据源不匹配问题 → 添加 `packName` + `resourcePath` 字段，服务端改用 StructurePacks 直接加载
- 确认弹窗"取消"改为调用 `cancelPlacement()` 回退到图鉴面板

**2. 放置动画 + 速度控制**（`CreativeStructureHandler`）
- `getStepsPerCall()` 从 1000 → 50 → **8** blocks/tick（~160 块/秒），肉眼可见逐块效果
- 添加 break 粒子效果（`levelEvent 2001`）

**3. 位置调整系统**（`MayorBookScreen`）
- WASD 微调（nudgeX/Y/Z）、QE 旋转（4 方向）、RF 高度调整
- RTS 摄像机偏航角计算方向，而非 player.getDirection()
- Tab 键切换 CAM（视角）/ NUDGE（微调）模式
- Ghost Preview 实时同步位置/旋转

**4. 体素预览修复**（`BuildingCatalogTab`）
- 移除 `isoScaleX` 的 4f 硬上限和 ×0.5 衰减，改用预览区域自适应
- `blockSize` 下限 1→2，阴影改为"高光白+阴影黑"双面
- 缩略图优先加载真实蓝图渲染，回退简化色块

**5. 真 3D 预览（Ghost Preview）**（`MetroGenesis.java` + `MayorBookScreen`）
- 注册 structurize 的 ClientEventSubscriber（反射）→ RenderLevelStageEvent 管线
- `updateGhostPreview()` / `removeGhostPreview()` 管理 RenderingCache
- 透明度 25%，serverSyncEnabled=false 避免网络同步
- 透视投影 + MC 原版 `BlockRenderer.renderSingleBlock()` 逐块渲染（比等轴测更真实）

**6. CatalogPlacementScreen（已删除）**
- 独立放置 Screen 流程：关闭管理书 → 自由走动 → 微调 → 放置
- 后改为 RTS 内放置（不关闭管理书，右键拖拽自由视角），删除该文件

**7. 放置状态持久化**
- `hasPendingPlacement` 标志 → 重开管理书自动恢复 placementMode + Ghost Preview

**8. 3D 预览居中 + 鼠标交互**
- Y 轴从 `-bpSizeZ/2` → `-bpSizeY/2`，`maxDim` 统一缩放
- 左键拖拽平移 + 滚轮缩放预览区

**9. 图鉴编辑功能**
- 文件夹树底部 [+] [✏] [✗] 工具栏 → 创建/重命名/删除文件夹和文件
- 文本输入对话框 + 删除确认对话框
- 文件系统操作（`Files.createDirectories/move/delete`）

**10. 三项修复（20:38）**
- AD 方向修正（交换正负号）
- 移除 UI 3×3 按钮网格
- Tab 模式切换 + 预览区右键拖拽旋转（previewRotX/Y）

**11. 城市需求面板**（`CityDemandsData` + `SyncCityDataMessage` + UI）
- 数据模型：人口/财政/建设/经济/短缺
- 网络同步：C2S 请求 → S2C 响应
- Tab 切换（图鉴/需求）、核心指标卡片、经济供需表格、短缺列表

**12. 全面交互修复（22:50）**
- 移除 Zone 功能区全部 UI（底部工具栏、键盘快捷键、鼠标点击）
