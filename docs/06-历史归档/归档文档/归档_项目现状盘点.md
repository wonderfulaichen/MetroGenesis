## 0. 一句话结论

**内容（蓝图）充足、放置管线成熟、经济/部门代码写得很完整——但"城市模拟"在运行时几乎没有运转**：
核心每日循环 `ColonyState.tick()` 从未被调用（其唯一调用方 `DailyScheduler` 被排除编译），
市民生成触发器被排除，job/workorder 被排除。
结果：一个**上了发条却没有指针、也不上发条的钟**。

---

## 1. 最关键的缺陷（必须最先修）

### 🔴 缺陷 A：每日模拟循环未接线（运行时死的）
- `ColonyState.tick(ServerLevel)` 是设计上的每日驱动：触发 `EconomyEngine.tick()` 与 `DepartmentManager.tick()`（5 局 `onDailyTick`）。
- 它的唯一调用方是 `colony/DailyScheduler.java`，而该文件在 `build.gradle` 第 129 行被 `exclude`。
- `MetroGenesis.onServerTick`（主类 295–321 行）只 tick 了：**道路**（`RoadPipelineController`）和 **structurize 撤销队列**（`Manager.onWorldTick`），**没有**调 `ColonyState.tick()`。
- **后果**：价格永不更新、税收永不收、科研永不推进。`EconomyEngine` 与 5 局全部"编译通过但运行时休眠"。

> 修复：在 `MetroGenesis.onServerTick` 的 `getAllLevels()` 循环里加 `ColonyState.get(world).tick(world)`；
> 或新建一个轻量 `DailyTickScheduler` 只做这件事。**不要**直接取消排除 `DailyScheduler`（它引用了同样被排除的 job/workorder，会编译失败）。

### 🔴 缺陷 B：市民永不生成（人口恒为 0）
- `MetroGenesisCitizen` 实体已注册（`MetroGenesis.java` 109–113 行），`colony/managers/CitizenManager` 已编译。
- 但市民**生成触发器** `citizen/CitizenEvents.java` 被排除编译；没有任何其他代码 spawn 市民。
- **后果**：`ColonyState.getPopulation()` 恒为 0 → 财政部税收恒为 0 → 经济循环虽然跑了也是空转。`ConstructionSite` 会 `assignNearbyUnemployed`（分配既有市民），但不会创造市民。

### 🔴 缺陷 C：生产/消费/物流无输入
- `core/economy/MarketData` 有 `recordProduction/recordConsumption/updateStock`，但调用方（`job/`、`workorder/`、`CitizenEvents`）全被排除。
- **后果**：市场有完整定价引擎，但无供需信号流入 → 价格恒定（引擎内有 `if 无活动则跳过` 的保护），等于没用。

## 6. 与文档的偏差（设计 ≠ 现实）

| DESIGN.md / ARCHITECTURE.md 描述 | 现实 |
|------|------|
| §3.4 完整每日结算（市民工作→产出→消费→供需→生育/死亡） | 实现类 `DailyScheduler`/`job`/`workorder` 全被排除，**未运行** |
| 5 大部门自动管理城市 | 仅框架+财政部/规划局逻辑；且因缺陷 A 全部休眠 |
| 市民系统（职业/技能/需求槽/行为） | 实体编译但**不生成**；AI(job)被排除 |
| 物流网络（无实体配送） | 代码存在性待核实，无生产输入则无意义 |

**含义**：文档描述的是"目标态"，不是"当前态"。计划须以"当前态"为起点，把 Dead Loop / 市民生成 / 生产输入 作为第一批要解的问题。

## 8. 给路线图的前提（见 PLAN.md）

1. **先修复缺陷 A/B/C**（让每日循环真跑、让市民能生成、让生产有输入）——否则后续 Zone 生长、RCI 都是空中楼阁。
2. Zone 生长（Phase 3 P0）应接入**建设局 `onDailyTick`**，并随缺陷 A 修复一并被驱动，而非在主类另起 tick。
3. 内容不缺，缺的是**分类映射**与**仿真接线**。
4. 任何改动**不能动** `build.gradle` 的嵌入式 4 包排除、`Constants.MOD_ID="minecolonies"`。
