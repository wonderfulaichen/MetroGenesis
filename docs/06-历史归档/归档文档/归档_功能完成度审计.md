> 审计日期：2026-07-08

> 范围：自研包 = `com.metrogenesis` 下**非嵌入**的部分。嵌入代码（minecolonies 2091 / structurize 222 / domumornamentum 226 / blockui 83）不在本表。

## 术语更正（重要）

用户于 2026-07-08 纠正：**不是"材质自定义"，是"方块自定义"**。
- ❌ 错误提法：道路"材质"硬编码。
- ✅ 正确提法：道路使用硬编码的**原版方块** `SMOOTH_STONE` / `STONE_BRICKS` / `DIRT`，而不是在 `init/` 注册过的**自定义方块**（如 road 专用方块）。这是"方块注册/方块自定义"缺口，**与贴图/模型 JSON（材质）无关**。

## 分系统详述

### 🟡 road（道路系统）
- ✅ **手动画路可用**：贝塞尔路径、点击放置、`commitSegment`/`placeRoad`/`rebuildNode`、撤销/重做均实现，是可玩核心路径。
- ❌ **自动连路管线是空桩**：
  - `road/handlers/PipelineRunner.java` 整文件为 `// TODO: implement actual road generation pipeline`，`runPipeline()` 是 no-op（注释标注 "Stub: placeholder for the pipeline execution logic"）。
  - `RoadPipelineController` 在 tick/区块加载时不断调用它，但**什么都不做**（扫描结构→A* 寻路→铺路 全未实现）。
- ❌ **方块自定义缺口（非材质）**：
  - `ROAD_SURFACE = SMOOTH_STONE`、`ROAD_CURB = STONE_BRICKS`、`ROAD_BASE = DIRT` 硬编码在 `RoadBuilder` / `RoadTemplate` / `RoadPlacer` / `RoadTemplateManager` 四处，未使用 `init/` 注册的自定义方块，也无配置项。
- ⚠️ **模板无玩家入口**：`RoadTemplateManager.setActiveTemplate` 全项目只有管理器自己调用；玩家没有切换直道/十字/自定义样式的 UI。`getAutoTemplate`（3+ 连接自动选十字）是否真正接到 `rebuildNode` 存疑。
- ⚠️ `RoadBlueprintBuilder` 多处 `return null`；`RoadPlacer` 疑似只取模板 Z=0 层。

### 🟡 citizen（市民系统）
- ✅ 数据模型/职业/Manager 齐：`citizen/types/CitizenData`、`citizen/job/{MerchantWorkGoal,FarmerWorkGoal,BuilderWorkGoal}`（均调用 `CitizenData.payToTreasury`）、`colony/managers/CitizenManager`、`colony/citizen/{CitizenNameFile,CitizenNameListener}`。
- ❌ **市民生成入口被排除**：`citizen/CitizenEvents.java` 在 `build.gradle` 第 142 行被 `exclude`，即 `EntityJoinLevelEvent` 自动生成市民的逻辑**不在构建内** → 市民实体从不生成，`population` 恒为 0。
- ⚠️ 冗余：旧 `citizen/CitizenData.java`（标注 `@deprecated` 重定向到 `citizen/types/CitizenData`）与后者并存，且各有 `.bak2` 备份文件。
- 直接后果：`payToTreasury` 永不触发 → 财政部（国库税收）无收入 → 经济无消费记录（缺陷 C 的一部分）。

### 🟡 colony（殖民地管理）

- ❌ **`colony/DailyScheduler.java` 被排除编译**（第 129 行）→ 原"每日调度器"本体未进入构建，R0 绕过了它直接 tick 经济/部门。

- `colony/citizen/`、`colony/managers/CitizenManager.java` 存在（编入）。

### 🟡 core/economy（C-Value 经济引擎）— 缺陷 C

## 与"仿真三缺陷"的对应关系

| 缺陷 | 表现 | 根因（本审计定位） |
|------|------|--------------------|
| A 每日循环没跑 | 经济/部门不推进 | **已修（R0 接线）**，`DailyScheduler` 因被排除由 `ColonyState.tick` 直接替代 |
| B 市民=0 | 国库无税收、经济无消费 | `CitizenEvents` 被排除 → 市民不生成（citizen 系统） |
| C 无产销输入 | 价格不动、经济空转 | 仅 `test/EconomySimulator` 调用市场 API，无游戏内生产者/消费者（core/economy + citizen） |

## 优先级建议（待用户拍板）

- **P0（让"城市活起来"可见）**：B（复活 `CitizenEvents` 生成市民）+ C（把农田/市民消费接入 `MarketAccess`，让产销真正灌入市场）。
- **P1（道路补全）**：填 `PipelineRunner` 自动连路；把硬编码原版方块改为 `init/` 注册的自定义方块（**方块自定义，非材质**）；给 `RoadTemplateManager` 加玩家切换入口。
- **P2（去孤儿/接线）**：复活市政厅（`TownHallBlock`+菜单+界面）；恢复 `HologramSyncPacket` 同步；商务部价格干预；catalog 图标加载；`ZoneBuilder` 三处 TODO。
- **P3（清理）**：移除 `.bak2` 冗余、旧 `citizen/CitizenData` 弃用重定向；验证 `layergrid` 运行时。
