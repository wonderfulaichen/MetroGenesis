### 关于「思索 / Ponder」的设计定位（用户拍板：2026-07-08）

**Ponder（思索）= Create 团队出的"交互式游戏内教程/指南库"**，核心是"**3D 模型预览 + 分步注释叠层**"。它不是城建 UI 模组，但其**交互范式非常适合本工程的「图鉴建筑预览」**——用户明确认可此方向。

**定位（分层）**：
- ✅ **图鉴建筑预览 = 采用 Ponder 范式**：选中建筑后，用 `hologram/` + `BlueprintRenderer` 渲染该建筑蓝图为可旋转 3D 预览，配分步注释（造价 `materialCost`、占地 `size`、可升级等级 `levels`、分类 `category`、设施类型 `buildingType`、功能说明）。→ 复用 `BuildingCatalogEntry` 已有元数据 + `StructurePacks.getBlueprint(packName, path)` 加载真实蓝图。
- ✅ **整体界面 chrome（顶栏/面板/按钮）仍照 `UI重建设计规范_v1.md`（Create 工业风 + Playlist 玻璃拟态, v1.2 配色）**——Ponder 提供的是"预览+注释"的交互灵感，不是整体皮肤。
- ❌ 不抄 Ponder 的教程流程引擎本身（我们已有 `BlueprintRenderer` + `BuildingCatalogEntry`，不必引入 Ponder 依赖）。

## 🔄 核心玩法循环（端到端 ①→⑨，2026-07-08 用户拍板）

城市从 0 到自运转的完整闭环，是评判"模拟城市"是否成立的标尺：

```
① 打开市长管理书（城市指南） → 选择风格包 → 命名城市（并确立归属）
   ↓
② 招人    CitizenEvents 生成 MetroGenesisCitizen → 注册进 CitizenManager（人口 > 0）
   ↓
③ 规划    玩家画 Zone（construction/Zone）→ 从 Catalog（建筑图鉴）选一栋建筑
   ↓
④ 放样    蓝图全息预览（hologram）→ 确认放置 ConstructionSite
   ↓
⑤ 建造    ZoneBuilder 从国库扣 C-Value、按风格包取建筑条目 → 分配市民去建造
   ↓
⑥ 上岗    建筑内部有对应的设施方块 → 市民被分配去工作
   ↓
⑦ 产销    农民收获 → MarketAccess.recordProduction；市民消耗 → recordConsumption
   ↓
⑧ 经济    ColonyState.tick(R0) → EconomyEngine 按供需调价 + DepartmentManager 每日结算 + 财政部收税入国库
   ↓
⑨ 再投资  国库 C-Value 反哺 ③⑤（造更多房）→ 更多市民 → 回到 ②
```

### 各环节当前状态（对照各 Phase）
| 步 | 环节 | 状态 | 说明 |
|----|------|------|------|
| ① | 命名城市 | ✅ | 市长书右侧城市概览加 ✎ 编辑按钮 → 输入框 + 确认/取消；发 `FoundCityMessage`(网络 id=4) → 服务端 `foundCity`/`setCityName`（未开城则开城，已开城则重命名）。命令 `/foundcity` 仍保留 |
| ② | 招人 | ✅ | `CitizenManager.onColonyTick` 开城后生成市民（R1 已接线） |
| ③ | 规划 Zone | 🔧 | Zone 绘制已实现；选建筑接 Catalog（图鉴 3D 预览已落地） |
| ④ | 放样 | 🟢 | 全息预览 + ConstructionSite 放置管线已具备 |
| ⑤ | 建造 | 🔧 | ZoneBuilder + C-Value 扣除（B1–B5 接线中，Phase 3 P0） |
| ⑥ | 上岗 | ❌ | 建筑设施方块→市民工作分配未接（依赖 job/workorder，现被排除编译） |

## R2：经济/生产输入（缺陷 C）✅（2026-07-08 完成，BUILD SUCCESSFUL）

**目标**：让 `MarketData` 收到产/销信号，价格随供需波动。

**根因（已定位）**：`MarketData.recordProduction/recordConsumption` 实现完整，但自研玩法代码中从无调用方（`MarketAccess.purchase/sell` 虽已实现却无人调用）→ 市场永远空转，价格恒为 `CValueRegistry` 基准值。

**做法（2026-07-08 实现）**：
- 新建 `core/economy/EconomySignals.java`：把"城市运转"翻译为供需事件。
- `ColonyState.tick()` 的每日闸门（`tickCounter % Config.settlementInterval`，与经济引擎日结算同节奏）调用 `EconomySignals.recordDailyColonyActivity(this, level)`（仅开城后 `hasTownHall()` 才运转）。
- 信号内容：
  - **市民消费（需求）**：每名市民每日消费 面包/小麦/苹果 各 1（经 `MarketAccess.purchase` 记 `recordConsumption`）。
  - **建筑产出（供给）**：每栋**已建成**建筑按类型产出——农场→小麦×5+面包×3；工坊→圆石×4+铁锭×1；仓库→箱子×1（经 `MarketAccess.sell` 记 `recordProduction`）。
  - **建筑维护（需求）**：住宅→原木×1+石头×1；市政厅→原木×2+石头×1（经 `MarketAccess.purchase` 记消耗）。
- 建筑来源：新增 `ConstructionManager.getAllSites()`（含已建成工地），`EconomySignals` 遍历并筛 `isCompleted()`。

**效果**：人口多于农场 → 食物涨价；工坊产出多于维护消耗 → 建材降价。市场首次出现真实供需波动，经济日报（TOP5 价格变化）开始有内容。

**验收**：开城→建农场/住宅→过数日 → `/cvalue` 显示小麦/面包/圆石/铁锭等价格偏离基准值；经济日报弹出波动提示。

**已知边界（非缺陷）**：产出/消耗系数为每建筑每市民固定值（RCI 轻量起步），未接入真实库存/物流（Phase 4）；商业区/工业分类（CAT_COMMERCIAL 映射）属 Phase 3 P1，不影响本 R2。
