## 0. 一句话结论

仿真"大脑"（经济 / 部门 / 殖民地 / 市民 / 建设）已是**原生自研、零嵌入依赖**——这是消化最成功的部分；但**运行时几乎没运转**：每日循环 / 市民生成 / 产销输入三大断点（缺陷 A/B/C）长期休眠，形成"上了发条却没指针、也不上发条的钟"。内容（蓝图）充足、放置管线成熟，**瓶颈在仿真接线与内容分类（RCI 映射），不在可用资产**。

## 1. 仿真三缺陷（核心断点，三份文档共同指向）

### 🔴 缺陷 A：每日模拟循环未接线（运行时死的）
- `ColonyState.tick(ServerLevel)` 是设计上的每日驱动：触发 `EconomyEngine.tick()` 与 `DepartmentManager.tick()`（5 局 `onDailyTick`）。
- 唯一调用方 `colony/DailyScheduler.java` 被 `build.gradle` 排除。
- `MetroGenesis.onServerTick` 只 tick 了道路（`RoadPipelineController`）与 structurize 撤销队列（`Manager.onWorldTick`），未调 `ColonyState.tick()`。
- **后果**：价格 / 税收 / 科研全休眠。**修复（R0，已接线）**：在 `onServerTick` 的 `getAllLevels()` 循环加 `ColonyState.get(world).tick(world)`；`DailyScheduler` 因被排除由 `ColonyState.tick` 直接替代。**不要**直接取消排除 `DailyScheduler`（它引用同样被排除的 job/workorder，会编译失败）。

### 🔴 缺陷 B：市民永不生成（人口恒 0）
- `MetroGenesisCitizen` 已注册、`colony/managers/CitizenManager` 已编译，但**生成触发器** `citizen/CitizenEvents.java` 被排除 → 无任何代码 spawn 市民。
- **后果**：`getPopulation()` 恒 0 → 财政部税收恒 0 → 经济空转。

### 🔴 缺陷 C：生产 / 消费 / 物流无输入
- `core/economy/MarketData.recordProduction/recordConsumption/updateStock` 的**唯一调用方**是 `core/economy/test/EconomySimulator.java`（开发测试桩），无游戏内生产者 / 消费者。
- `SyncCityDataMessage` / `CValueCommand` 只**读**不写。
- **后果**：`todayProduced/Consumed` 恒 0 → `adjustPricesBySupplyDemand` 因 `if (todayConsumed==0 && todayProduced==0) continue` 跳过全部物品 → 价格永不动、国库无收入。

## 8. 建议动手顺序与优先级

### 执行铁律（顺序不可乱）
R0 唤醒每日循环 → R1 市民生成 → R2 经济输入 → Phase 3 P0 区域生长接线。

### 三档方案（整合自消化进度审计 + 功能完成度优先级）
- **①（推荐，已部分执行）先让"自己的大脑"转起来**：R0 唤醒每日循环（✅ 已做）→ R1 市民生成入口 → R2 经济产销输入 → 修 `BlueprintLibraryData` 孤儿。理由：仿真已是自有代码，接线成本最低、最贴"做自己的模组"。
- **② 先减负（剥离死重，不含 domum）**：核实 structurize 对 minecolonies 依赖面 → 把 minecolonies 降为"仅常量 + 蓝图资源"最小依赖。短期不增可玩性，有破坏风险。
- **③ 先重分类内容（内容自主）**：补全 `CategoryMapper`（尤其商业/工业映射）→ 25 风格包重分类为 RCI → 接 Zone 生长让分类可见（依赖 R0/R1 先跑）。

### 自研功能优先级（整合）
- **P0（让"城市活起来"可见）**：B（复活 `CitizenEvents` 生成市民）+ C（把农田/市民消费接入 `MarketAccess`，让产销真正灌入市场）。
- **P1（道路补全）**：填 `PipelineRunner` 自动连路；方块自定义（注册 road 专用方块）；`RoadTemplateManager` 加玩家切换入口。
- **P2（去孤儿/接线）**：复活市政厅（`TownHallBlock`+菜单+界面）；恢复 `HologramSyncPacket` 同步；商务部价格干预；catalog 图标加载；`ZoneBuilder` 三处 TODO（扣款/tile/风格包）。
- **P3（清理）**：移除 `.bak2` 冗余、旧 `citizen/CitizenData` 弃用重定向；验证 `layergrid` 运行时。

### 不可触碰约束
任何改动**不能动** `build.gradle` 的嵌入 4 包排除、`Constants.MOD_ID="minecolonies"`、嵌入包名。
