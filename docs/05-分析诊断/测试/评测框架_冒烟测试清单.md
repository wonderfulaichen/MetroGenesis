## ① 建筑工具 (Build Tool) — P0 必须测试

### P0-01 🔴 右键地面打开建筑选择列表
- **测试场景**：手持 BuildingToolItem，对任意非空气方块右键
- **预期结果**：打开 BuildToolScreen，列表显示所有已启用（enabled）的建筑类型
- **测试方法**：创造模式 → give 建筑工具 → 对草方块右键
- **边缘情况**：对不透明方块 / 透明方块 / 液体右键
- **失败影响**：🔴 阻断 — 整个建造系统失去入口

### P0-02 🔴 选中建筑后点击"确认"发送放置请求
- **测试场景**：在 BuildToolScreen 中选择一个建筑类型，点击确认按钮
- **预期结果**：客户端发送 `BuildToolPlacementMessage`，服务端开始放置逻辑
- **测试方法**：打开 GUI → 选择"市政厅" → 确认 → 观察服务端 console 日志

### P0-03 🔴 旋转/镜像按钮功能
- **测试场景**：BuildToolScreen 中有 R/L 旋转按钮和镜像按钮
- **预期结果**：点击后 `currentRotMir` 状态正确切换（NONE → ROTATE_90 → ROTATE_180 → ROTATE_270 等）
- **测试方法**：打开 GUI → 点击旋转按钮 4 次 → 确认回到初始状态 → 点击镜像 → 确认状态变化
- **失败影响**：🔴 阻断 — 建筑朝向无法控制

### P0-04 🟡 建筑列表为空时的表现
- **测试场景**：所有 BuildingType 的 `isEnabled()` 返回 false（如配置禁用）
- **预期结果**：GUI 正常打开，显示"无可用的建筑类型"提示，确认按钮禁用
- **测试方法**：临时修改配置 → 打开建筑工具 → 确认不会崩溃
- **影响评估**：极端情况，但空列表崩溃属于阻断级别

### P0-05 🟡 建筑放置后的二次确认反馈
- **测试场景**：确认放置后，客户端收到服务端回应
- **预期结果**：客户端显示"建筑已加入建造队列"或类似反馈，不静默
- **测试方法**：放置建筑 → 观察聊天栏 / HUD / GUI 是否有反馈
- **影响评估**：🟡 困惑 — 无反馈会让玩家以为操作失败

## ② 选框系统 (Zone Selection) — P0 必须测试

### P0-06 🔴 选区起点→终点框选流程
- **测试场景**：使用扫描工具/物品在俯视视角设定选区
- **预期结果**：客户端实时渲染蓝色半透明选区，角标显示尺寸
- **测试方法**：拿出扫描工具 → 左键设起点 → 拖动 → 右键设终点 → 观察选区渲染

### P0-07 🔴 64×64 面积限制
- **测试场景**：框选超过 64×64 的区域
- **预期结果**：选区被截断或提示"区域过大"（不超过 4096 方块）
- **测试方法**：框选 65×65 → 确认截断或提示
- **边缘情况**：64×64 正好 → 应该通过；64×1 窄条 → 应该通过

### P0-08 🟡 选区悬停高亮
- **测试场景**：鼠标悬停在已框选的方块上
- **预期结果**：该方块边缘高亮或选区整体亮度变化，与未悬停时有视觉区分
- **测试方法**：框选一个 10×10 → 鼠标在选区内移动 → 观察视觉反馈

### P0-09 🟡 跨维度选区（地表→地下）
- **测试场景**：起点在山顶（Y=100），终点在地下（Y=20）
- **预期结果**：选区正确包含 Y 轴方向的所有方块，实时预览体积正确
- **测试方法**：从高处向低处框选 → 观察选区是否包含全部体积
- **影响评估**：🟡 困惑 — 地形不平场景常见，Y 轴遗漏会导致保存的蓝图不完整

## ③ 市长书 (MayorBook GUI) — P0 必须测试

### P0-10 🔴 GUI 基本打开与关闭
- **测试场景**：手持市长书右键打开 / 按 ESC 关闭
- **预期结果**：窗口打开，游戏世界背景可见（透明背景），不暂停游戏
- **测试方法**：手持 MayorBookItem 右键 → 打开 GUI → 按 ESC → 回到游戏

### P0-11 🔴 3D 城市鸟瞰视角
- **测试场景**：MayorBookScreen 中的虚拟相机控制
- **预期结果**：WASD 平移、右键拖拽旋转、滚轮缩放，视角平滑跟随
- **测试方法**：打开市长书 → WASD 测试平移 → 右键拖拽 → 滚轮拉近/拉远

### P0-12 🟡 底部工具栏区域分类切换
- **测试场景**：点击底部 5 个分类按钮（住宅 / 工业 / 商业 / 农业 / 公共）
- **预期结果**：选中状态切换，3D 视角中对应颜色标记的区域高亮
- **测试方法**：点击"住宅" → 确认按钮高亮样式改变 → 点击"工业" → 确认切换

### P0-13 🟡 建筑图鉴分类浏览 + 目录模式
- **测试场景**：市长书切换到"目录模式"，浏览已保存的标准化蓝图
- **预期结果**：按分类展示蓝图列表，点击后显示 3D 预览
- **测试方法**：打开目录模式 → 选择"住宅"分类 → 浏览列表 → 点击一个蓝图 → 观察 3D 预览

### P0-14 🔵 放置确认流程
- **测试场景**：在市长书中选择一个蓝图并点击"放置到世界"
- **预期结果**：切换到 BuildToolScreen 或直接弹出位置选择模式
- **测试方法**：市长书 → 选中蓝图 → 放置 → 在地面点击 → 完成放置
- **建议**：市长书 3D 预览和建筑工具放置之间应有平滑的"预览→确认→放置"过渡

## ④ 道路系统 (Road System) — P0 必须测试

### P0-15 🔴 道路起点→终点设置
- **测试场景**：使用道路工具左键设起点 A，右键设终点 B
- **预期结果**：地面显示两点标记，两点间出现贝塞尔曲线预览
- **测试方法**：手持道路工具 → 左键点地面 → 右键点远处地面 → 观察预览线
- **失败影响**：🔴 阻断 — 道路铺设的第一步

### P0-16 🔴 贝塞尔曲线曲率调整
- **测试场景**：拖拽控制点调整曲率
- **预期结果**：曲线实时更新，曲率值从 -1.0 平滑渐变到 1.0
- **测试方法**：设好 A、B 点 → 拖拽控制点 → 观察曲线形态变化 → 拖回直线状态
- **失败影响**：🔴 阻断 — 不能转弯的路毫无意义

### P0-17 🟡 纯直线道路（曲率=0）
- **测试场景**：曲率设为 0，确认铺设
- **预期结果**：道路沿两点间最短路径铺设，无弯曲
- **测试方法**：设 A、B → 曲率归零 → 确认铺设 → 拆掉观察路面

### P0-18 🟡 坡度适应（地形跟随）
- **测试场景**：在丘陵/山地地形铺设道路
- **预期结果**：道路自动贴合地形坡度，不悬空、不穿模
- **测试方法**：在起伏地形设 A、B（有高差）→ 铺设 → 观察路面是否贴合地面

### P0-19 🔵 交叉口自动检测与模板对齐
- **测试场景**：铺设两条交叉的道路
- **预期结果**：第二条道路在交点处自动匹配交叉口模板
- **测试方法**：铺一条南北路 → 铺一条东西路 → 观察交点
- **备注**：需甲方确认交叉口模板是否已实现

## ⑤ 区域规划 (Zone Grid) — P0~P1

### P0-20 🟡 区域 hover 高亮
- **测试场景**：鼠标在 1×1 颗粒度网格上移动
- **预期结果**：当前悬停的单元格有颜色高亮反馈，与相邻格可区分
- **测试方法**：打开区域规划模式 → 在网格上移动指针 → 观察高亮跟随

### P0-21 🔴 降采样渲染性能
- **测试场景**：大范围区域（如 100×100）的网格渲染
- **预期结果**：帧率无明显下降（需甲方实测帧率数据）
- **测试方法**：框选 100×100 区域 → 移动视角 → 评估卡顿
- **备注**：需甲方实测 FPS

### P0-22 🔵 颗粒度切换测试
- **测试场景**：在 1×1 精细模式和 4×4 粗略模式之间切换
- **预期结果**：网格密度变化，选区和标记正确映射
- **测试方法**：切换颗粒度 → 框选同样区域 → 确认结果正确
- **建议**：如果只有 1×1 一种模式，此项跳过

## ⑥ 殖民地系统 — P0 必须测试

### P0-23 🔴 殖民地创建（放置市政厅）
- **测试场景**：使用建筑工具放置"市政厅"
- **预期结果**：市政厅方块放置完毕 → 系统自动创建 ColonyState → 弹出成功提示
- **测试方法**：工具放市政厅 → 等待建造完成 → 观察聊天栏
- **失败影响**：🔴 阻断 — 没有殖民地什么都没法玩

### P0-24 🔴 市民生成
- **测试场景**：市政厅建造完成后
- **预期结果**：自动生成 3 名市民，显示在殖民地人口计数中
- **测试方法**：建造市政厅 → 观察周围是否有市民实体 → 打开市长书查看人口
- **边缘情况**：空置域/超平坦/下界→生成位置是否合理

### P0-25 🔴 建筑放置 GUI 交互（ConstructionMarkerScreen）
- **测试场景**：右键点击 ConstructionMarkerBlock
- **预期结果**：打开 ConstructionMarkerScreen，显示可建造列表，包含进度条
- **测试方法**：放一个施工标记 → 右键 → 检查 GUI 内容和互动

### P0-26 🟡 殖民地跨存档持久化
- **测试场景**：创建殖民地 → 保存退出 → 重新进入存档
- **预期结果**：殖民地状态恢复（建筑、人口、资金），不丢失数据
- **测试方法**：建造市政厅 + 一个建筑 → 保存退出 → 重进 → 检查
- **影响评估**：🟡 困惑 — 重新进存档发现殖民地消失是最常见的弃坑原因

## ⑦ GUI 交互 — 各关键界面走查

### P0-27 🟡 ConstructionMarkerScreen — 进度条显示
- **测试场景**：有正在施工的建筑时打开施工管理界面
- **预期结果**：进度条正确反映施工完成百分比
- **测试方法**：放置一个建筑 → 立即打开施工管理 → 观察进度条
- **影响评估**：进度条是玩家判断"还有多久"的唯一依据

### P0-28 🔴 BlueprintCaptureResultScreen — 扫描结果保存流程
- **测试场景**：框选一个区域 → 触发扫描 → 弹出保存对话框
- **预期结果**：显示 3D 预览、方块数、尺寸；可输入名称、旋转/镜像、保存
- **测试方法**：框选 10×10 区域 → 触发扫描 → 在对话框中输入名称 → 保存

### P0-29 🟡 保存命名冲突处理
- **测试场景**：保存同名蓝图
- **预期结果**：提示"名称已存在"并询问是否覆盖
- **测试方法**：保存一个"test" → 再保存一个"test" → 观察弹窗
- **影响评估**：🟡 困惑 — 静默覆盖可能让玩家丢失之前的扫描成果

### P0-30 🔵 建筑工具 GUI — 配色一致性
- **测试场景**：对比 BuildToolScreen 与 MayorBookScreen 的视觉风格
- **预期结果**：色彩体系一致（深色+蓝紫强调色 VS 磨砂玻璃灰调）
- **测试方法**：先后打开两个 GUI → 对比配色、字体、边框
- **建议**：BuildToolScreen 与 MayorBookScreen 在不同文件中有不同的调色板定义，建议统一主题

## 玩家第一印象总评

### 新手体验路径

```
进入世界 → 拿建筑工具 → 放市政厅 → 等市民生成 → 规划城区 → 铺路 → 建建筑
```

### 可能卡住的节点

| 节点位置 | 风险等级 | 原因 |
|---------|:-------:|------|
| 找不到建筑工具 | 🟡 | 创造标签页是否只有一个 MetroGenesis 统一标签？ |
| 不知道建市政厅是第一步 | 🟡 | 没有引导/任务系统 |
| 铺路时控制点拖拽不顺畅 | 🔵 | 缺少操作提示文字 |
| 建筑放下去没有反馈 | 🔴 | 服务端处理完成后是否通知客户端？ |
| 不知道如何在市长书中放置建筑 | 🟡 | 从目录预览到实际放置的过渡是否明确 |

### 需甲方实测项汇总

以下项超出纯黑盒逻辑走查范围，需实际运行后补充数据：

1. **降采样渲染性能**（P0-21）— 大范围 Zone Grid 帧率
2. **贝塞尔曲线实时预览帧率** — 拖拽控制点时是否卡顿
3. **全息蓝图渲染覆盖范围** — 超过 32 格是否闪烁/消失（MC 原版渲染距离问题）
4. **道路模板在转角处的纹理衔接** — 肉眼观察是否有接缝
5. **市民寻路碰撞** — 多个市民是否围堵在建筑门口
6. **市长书 3D 视角的 FOV 变化** — 滚轮缩放是否平滑
