# Phase 5 — 运行时测试向导

> 版本：0.0.1 | 目标：Minecraft Forge 1.20.1 | jar: `build/libs/metrogenesis-0.0.1.jar`

---

## 前置准备

1. 启动 **Minecraft 1.20.1 Forge 客户端**（本模组无外部依赖，所有嵌入模组已内嵌，单 jar 即可运行）
2. **创建一个新的超平坦世界**（或创造模式现有世界），推荐超平坦避免地形干扰测试
3. 确认模组已加载：游戏主菜单 → Mods 按钮 → 搜索 "MetroGenesis"

---

## P0 — 核心功能测试（按顺序执行）

### 测试 0.1 — `/cvalue` 命令

```
操作：
  1. 进入游戏，按 T 打开聊天框
  2. 输入 /cvalue 按 Enter

预期结果：
  ✅ 聊天框显示类 SimCity 城市价值信息（C-Value 指标，含人口/满意度/资金等）
  ✅ 命令执行不报错、不抛异常
  ❌ 显示 "Unknown command" → 说明命令未注册

如果失败：
  - 确认 MetroGenesis 模组已加载（Mod 列表中有 "MetroGenesis"）
  - 检查日志中是否有 CommandRegistrationException
```

### 测试 0.2 — 市长管理书打开 + RTS 相机

```
操作：
  1. 手持市长管理书（创造模式物品栏 MetroGenesis 标签页 → "市长管理书"）
     或通过 /give @s metrogenesis:mayor_book 获取
  2. 右键使用

预期结果：
  ✅ 游戏不暂停，屏幕变暗（玻璃态磨砂效果）
  ✅ 玩家视角飞到高空，俯视地形
  ✅ 屏幕顶端出现状态栏（城市名/资金/人口/满意度）
  ✅ 底部出现工具栏（住宅/工业/商业/农业/公共按钮 + R 道路 + T 图鉴）
  ✅ WASD 移动相机、滚轮缩放、Q/E 旋转
  ✅ ESC 恢复正常视角和位置

如果失败：
  崩溃 → 发崩溃日志给我
  黑屏 → 按 ESC 退出，检查 mayor_book 物品是否有效
  相机不动 → 确认在英文输入法下，WASD 在中文输入法不生效
```

### 测试 0.3 — 区块区域绘制

```
操作：
  1. 打开市长管理书
  2. 底部工具栏点击 "住宅" 按钮（或按键盘 1）
  3. 在 3D 鸟瞰视图上左键拖拽画一个矩形（~10×10 格）
  4. 松开左键

预期结果：
  ✅ 选中区域显示半透明绿色覆盖层（拖拽中实时预览）
  ✅ 松手后区域保持绿色染色（已确认）
  ✅ 鼠标悬停方块显示白色线框高亮
  ✅ 按 ESC 或关闭管理书时区域不消失

如果失败：
  区域不显示 → 检查 ZoneRenderer 是否注册（查看日志 RenderLevelStageEvent 相关错误）
  拖拽没反应 → 确认先点击了分区类型按钮（住宅/工业等）
```

### 测试 0.4 — 道路绘制

```
操作：
  1. 打开市长管理书
  2. 按 R 键进入道路模式（底部 "R 道路" 按钮高亮）
  3. 在鸟瞰视图上左键拖拽一条直线（约 20 格长）
  4. 松手后看到半透明灰色方块投影（待确认状态）
  5. 右键确认施工

预期结果：
  ✅ R 键切换道路模式，按钮高亮
  ✅ 拖拽时显示灰色半透明方块预览
  ✅ 松手后预览转为暖灰色（待确认投影）
  ✅ 右键确认后，预览变为实心方块（SMOOTH_STONE / STONE_BRICKS 路面）
  ✅ 退出管理书，回到地面能看到铺设的道路

如果失败：
  投影不显示 → 移动一下位置再试。如果仍然不显示，检查 RoadRenderer 日志
  确认后道路没出现 → 可能在服务端执行出错，查看 latest.log 中 road 相关错误
  SMOOTH_STONE 不是想要的外观 → 当前材质硬编码，Phase 5 后续会做模板选择 UI
```

### 测试 0.5 — 建筑蓝图放置

```
操作：
  1. 打开市长管理书
  2. 按 T 键打开蓝图图鉴面板
  3. 左侧分类点击任意分类（或保持 "全部"）
  4. 右侧列表点击一个蓝图选中
  5. 右下角 3D 预览区显示该蓝图的等轴测体素渲染
  6. 底部点击 "放置" 按钮
  7. 管理书消失，回到自由 RTS 视角
  8. 左键点击地面确定放置位置
  9. 弹出确认弹窗 → 点击 "确认"

预期结果：
  ✅ T 键打开图鉴面板（毛玻璃效果覆盖 3D 视图）
  ✅ 分类列表显示所有蓝图分类
  ✅ 选择蓝图后右下角显示体素预览（小立方体组合）
  ✅ 放置流程走通：T → 选择 → 放置 → 点击地面 → 确认
  ✅ 确认后蓝图放置到世界中

如果失败：
  图鉴面板是空的 → 你的世界还没有保存任何蓝图。需要先用蓝图扫描工具录制建筑
  体素预览不显示 → 检查 BlueprintLibraryData 是否可访问
  放置后建筑没出现 → 确认 NetworkHandler 正确发送了 BlueprintPlacementMessage
```

### 测试 0.6 — 区域持久化

```
操作：
  1. 完成测试 0.3，在某个位置画了一个住宅区
  2. 按 ESC 关闭管理书，回到正常视角
  3. 绕一圈看看地面上是否能看到区域染色（虽然从地面看可能不明显）
  4. 再次右键使用市长管理书
  5. 之前画的住宅区是否还在？

预期结果：
  ✅ 重开管理书后，之前画的区域仍然显示绿色染色
  ✅ 区域数据跨会话持久化

如果失败：
  区域丢失 → LayerGridSavedData 持久化可能有问题
  检查 world 目录下是否有 `data/metrogenesis_layergrid.dat` 文件
```

### 测试 0.7 — 建筑工具

```
操作：
  1. 打开聊天框，输入 /building_tool（创造模式物品栏也可以找到）
  2. 手持建筑工具
  3. 右键点击地面

预期结果：
  ✅ 获得名为 "建筑工具" 的物品（金色名称）
  ✅ 右键地面打开 MineColonies 建筑选择窗口（有分类列表 + 建筑预览）
  ✅ 窗口可正常关闭

如果失败：
  拿不到物品 → /building_tool 是假命令，用 /give @s metrogenesis:building_tool
  右键没反应 → 确认物品拿在手上，对着方块右键
  窗口是空的 → Structurize 的 BuildToolWindow 可能需要蓝图数据
```

## 第一轮测试指南

**建议顺序：**
1. 先测 0.1 → 0.2 → 0.3 → 0.4（核心交互链）
2. 再测 0.6 → 0.7（持久化和工具）
3. 最后 0.5（需要蓝图文件配合）

**遇到崩溃：**
按 Ctrl+C 复制崩溃窗口内容给我，或者发 `logs/latest.log` 的最后 100 行。

**遇到非崩溃异常（功能没反应）：**
打开聊天框看有没有报错信息。有些错误会在聊天框显示红色文字。
