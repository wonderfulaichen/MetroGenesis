## 七、关键发现与改造建议

### 🔴 痛点

1. **贴图完全缺失** — `textures/` 目录为空。两个工具的 item 模型引用不存在的贴图，游戏中显示为紫黑块
2. **住宅/工坊/仓库无自定义模型** — 直接用 `oak_planks` 占位，视觉效果差
3. **house/workshop/warehouse 的 BlockItem 已注册** — 但对应的功能逻辑（市民入住/生产/存储）尚未实现
4. **施工围栏粒子效果** — 代码已写但未在游戏中验证效果
5. **MineColonies 覆盖资源冗余** — ~90 个 blockstates + ~200 张贴图全部保留，未来清理 MineColonies 依赖时需移除

### 🟡 可优化

1. **BlockUI 自研副本** — 当前无问题，但长期应考虑精简（移除未使用的 UI 控件），而非保留整个副本
2. **系统解耦良好** — 市民/施工/工单/日常结算各自独立管理类，模块化不错
3. **NBT 序列化覆盖** — CitizenData/ConstructionSite/WorkManager 都有完整 NBT，持久化可靠

### 🟢 建议优先级

| 优先级 | 事项 | 说明 |
|--------|------|------|
| P0 | **补贴图** | building_tool + scan_tool 缺失贴图 → 否则游戏内紫黑块 |
| P1 | **house/workshop/warehouse 模型** | 至少给基础模型，不能一直用 oak_planks |
| P2 | **淘汰 MineColonies 源头文件** | 计划阶段的重要一步，释放 2000+ 文件 |
| P3 | **功能实现** | 住宅入住、工坊生产、仓库存储逻辑 |
| P4 | **精简 BlockUI 副本** | 移除未用控件 |
