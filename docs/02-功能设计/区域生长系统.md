# 区域生长系统设计 v2

> 版本：v2.0 | 最后更新：2026-07-08 | 状态：🔄 设计中（Phase 3 P0 待接线）
> 基于蓝图设施方块检测的区域→建筑自动生长系统
> 核心原则：蓝图有什么，系统就认什么，不预设固定尺寸/价格/类型

---

## 1. 区域参数

画区时玩家设定 3 个参数，决定区里长什么建筑：

| 参数 | 可选值 | 说明 |
|------|--------|------|
| `zoneType` | 住宅 / 商业 / 工业 / 农业 / 公共 / 混合 | 决定筛选哪些蓝图 |
| `density` | 低 / 中 / 高 | 控制覆盖率 + 建筑规模过滤 |
| `ownership` | 私人 / 公有 | 私有缴税，公有不缴税，从国库扣钱建造 |

UI 交互：进入区域模式 → 底栏显示一排参数设定
```
[🏠住宅][🏪商业][🏭工业][🌾农业][🏛公共][🔄混合]
  [○低密度][●中密度][○高密度]
  [□公有]
```

## 2. 蓝图扫描：建筑属性自动推导

每个蓝图在扫描时（已在 catalog 扫描流程中），增加以下推导：

```
Blueprint 文件 → 遍历 blockInfoList →
  ① 检测设施方块（instanceof AbstractBlockHut）
     → 确定建筑类型（住宅/农场/工坊/仓库/...）
  ② 记录占地尺寸（blueprint.getSizeX() × getSizeZ()）
  ③ 记录建筑高度（blueprint.getSizeY()）
  ④ 统计所有非空气方块的 C-Value 总和 → 造价
  ⑤ 检测是否同时包含"商业"和"住宅"设施方块 → 标记混合用途
```

扩展 `BuildingCatalogEntry` 新增字段：

```java
public record BuildingCatalogEntry(
    String name,
    String packName,
    String resourcePath,
    String category,          // MetroGenesis 分类
    String mcCategory,        // 殖民地原始目录
    BlockPos size,            // (width, height, depth)
    Set<Integer> levels,
    boolean hasIcon,
    // ═══ 新增 ═══
    String buildingType,      // hut block 名称（如 "farmer", "townhall", "warehouse"）
    boolean isMixedUse,       // 是否混合用途（含多个类型设施方块）
    long materialCost         // 材料总 C-Value
) {}
```

### 设施方块 → 区域类型映射

```
检测到设施方块                        → 该建筑属于
BlockHutFarmer / BlockHutCowboy      → 农业
BlockHutBaker / BlockHutTavern       → 商业
BlockHutTownHall / BlockHutSchool    → 公共
BlockHutHouse / fundamentals 目录    → 住宅
BlockHutBlacksmith / BlockHutSawmill → 工业
```

不依赖硬编码——把 `AbstractBlockHut` 的子类名称映射到 MetroGenesis 区域类型，配置在 `CategoryMapper` 中扩展即可。

## 3. 密度规则

密度控制两个维度：**覆盖率** 和 **建筑规模过滤**

```
低密度（Low）:
  coverage = 30%   // 填满区域 30% 面积即可
  filter: 只取 xz 面积 ≤ 36 的小建筑（适合独栋、小店）
  剩余 70% → 绿化/空地/花园

中密度（Medium）:
  coverage = 50%
  filter: xz 面积 16~64 的中等建筑（适合联排、公寓）

高密度（High）:
  coverage = 70%
  filter: xz 面积 ≥ 36 的大建筑（适合大厦、酒店、商场）
```

## 4. 装箱算法

### 4.1 相邻区合并

所有 `zoneType` 相同且共享完整边的 `ZoneRect` 在生长前先合并：

```
合并规则：
  两个 ZoneRect 的 zoneType 相同
  && 共享至少 1 格长的完整边
  → 合并为外框矩形（union of bounds）
  → 在合并后的外框上划格

可视化效果：
  ┌───A───┐┌───B───┐  →  ┌─────A+B─────┐
  │ 店│店 ││ 店│店 │     │ 店│店│店│店 │ ← 一整条街面
  └───────┘└───────┘     └────────────┘
```

### 4.2 建筑尺寸优先级

从 catalog 筛出的建筑按面积从大到小排序，优先放大的：

```
1. 按 xz 面积倒序排列可用蓝图
2. 沿 road-facing 边从一端开始试放
3. 每次取当前最大能塞进的蓝图
4. 塞到 coverage 目标达成
5. 剩余的格子填装饰（路灯/长椅/树）
```

### 4.3 Seamless 对齐

相邻建筑优先匹配 LEFT_WALL / RIGHT_WALL 类型；

```
沿街面排布：
  第 1 格：CORNER 或 LEFT_WALL
  中间格：LEFT_WALL 或 RIGHT_WALL
  最后格：CORNER 或 RIGHT_WALL

独立格：BOTH_WALLS
```

每个格子在匹配时，检查相邻格已选的蓝图 → 匹配对应的 SeamlessType。

## 5. 生长流程

```
1. 玩家画区 → ZoneRect(stage=PLANNING)
   → 放置 ConstructionTape 围栏
   → 等待玩家确认

2. 确认后 → stage=PENDING
   → ZonePlanner.plan() 执行：
     a) 检测相邻同类型区 → 合并外框
     b) 从 catalog 筛出符合 zoneType+density 的蓝图
     c) 按尺寸倒序 + 装箱算法 → 生成 BuildingSlot 列表
     d) 计算总造价（所有建筑的材料费之和）
     e) 检查 C-Value 预算 → 不够钱标记为"资金不足"
   → stage=BUILDING

3. stage=BUILDING
   → ZoneBuilder.execute() 逐 tick 放置
   → 每放完一个建筑扣掉对应材料 C-Value
   → 全部放完 → stage=COMPLETED

4. stage=COMPLETED
   → 建筑固定在世界中
   → 可触发升级（level up）/ 地价变化后替换
```

### 挂起条件

```
- 资金不足 → 停留在 PENDING，等 C-Value 够了自动恢复
- 用地不足（区域内已有建筑占位）→ 跳过占位格继续
- 蓝图加载失败 → 跳过该格，记录日志
```

## 6. 混合用途

混合区（zoneType=mixed）不限制设施方块类型，任何类型的蓝图都可以长在这里。

选择蓝图时按以下倾向排序：
```
1. 临街格 → 优先选含商业设施方块的蓝图
2. 内部格 → 优先选住宅/其他设施方块的蓝图
3. 如果一个蓝图同时包含商业+住宅设施方块 → 优先放临街格
```

---

## 7. 公有建筑

`ownership=state` 的区：
- 不区分 density，只放公共建筑（TownHall, School, Hospital 等）
- 从国库直接扣 C-Value，不收税
- 区内蓝图仅匹配含公共设施方块的建筑

---

## 8. 未来扩展：地价系统

地价暂不实现，但预留接口。规划中的地价影响：

```
每个格子的地价 = 基础地价 + 道路加成 + 公园加成 - 污染扣减
建筑 minLandValue <= 格子地价 → 允许生长
地价不足时 → 该格等待地价提升后生长
```
