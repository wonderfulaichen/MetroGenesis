## 1. 根因分析

### 问题链

```
用户点击"职业切换" → TownHallMenu.clickMenuButton()
  → CitizenData.setJob(job)          ✅ 写数据
  └→ 不写实体 SynchedEntityData     ❌ 断裂
  └→ 不重挂 AI Goal                ❌ 断裂

市民生成 → CitizenEvents.onEntityJoin()
  → ModJobs.createGoal() + addGoal()  ✅ 挂载一次
  └→ 此后职业变更不复触发           ❌ 只有一次

设施建成 → ConstructionSite.complete()
  → assignNearbyUnemployed()
    → switch-case 硬编码             ❌ WAREHOUSE → null
    → addGoal(1, ...)                ❌ 与 LookAtPlayer 同 priority
```

### 对照 MineColonies 的差距

| 方面 | MineColonies | PolisCraft (修复前) |
|------|-------------|-------------------|
| AI 挂载时机 | Job 每 tick 检查变更，`createAI()` 工厂模式 | 只在 `EntityJoinLevelEvent` 挂载一次 |
| 职业↔实体同步 | `Job.View` 客户端同步 + `EntityCitizen.setJob()` | `SynchedEntityData` 和服务端 `CitizenData` 不同步 |
| 建筑↔职业映射 | `BuildingEntry` 注册时关联，`getJobName()` 方法 | 硬编码 `switch-case` 于 `CitizenManager` |
| Goal 优先级 | 状态机独立管理 | 与 `LookAtPlayerGoal` 共用 priority 1 |

## 2. 修复清单

### 2.1 `PolisCraftCitizen.java` — 新增统一 AI 重挂方法

**改动**:
- 新增 `currentJobGoal` 字段 (`Goal`) — 追踪当前职业 AI
- 新增 `reloadJobAI()` 方法 — 原子替换：`removeGoal(old) + createGoal(new) + addGoal(1, new)`
- `setCitizenJob()` 末尾自动调用 `reloadJobAI()`

**效果**: 任何时候职业变更（GUI 切换 / 分配）都会自动重挂 AI，不再只依赖 `EntityJoinLevelEvent`。

### 2.2 `CitizenEvents.java` — 简化为委托

**改动**:
- `onEntityJoin()` 改为调用 `citizen.reloadJobAI()`

**效果**: 统一入口，消除重复维护 `addGoal` 代码。

### 2.3 `TownHallMenu.java` — 补写实体同步（核心修复）

**改动**:
- 新增 `PolisCraftCitizen`、`Entity` 导入
- `clickMenuButton()` 中职业变更块：在 `data.setJob(newJob)` 之后
  - `findEntityByUUID(data.getUUID())`
  - `citizen.syncFromData(data)` — 写回 SynchedEntityData
  - `citizen.reloadJobAI()` — 重挂 AI
  - 实体不在世界时打 warn 日志

**效果**: 修复「设施不能选择建筑师」问题 — 职业切换现在完整同步到实体。

### 2.4 `BuildingType.java` — 数据驱动的职业映射

**改动**:
- 新增 `requiredJobId` 字段 (`String`)
- 新增 9 参构造函数（保留旧 8 参向后兼容）
- 所有预置建筑增加 `requiredJobId`：
  - `TOWN_HALL` → `null`（不分配）
  - `FARM` → `"farmer"`
  - `HOUSE` → `null`（不分配）
  - `WORKSHOP` → `"builder"`
  - `WAREHOUSE` → `"merchant"`
- 新增 `getRequiredJobId()` getter

### 2.5 `CitizenManager.java` — 使用 BuildingType 映射

**改动**:
- `assignNearbyUnemployed()` 硬编码 `switch-case` 替换为：
  ```java
  BuildingType buildingType = BuildingType.fromId(buildingTypeId);
  String jobId = buildingType.getRequiredJobId();
  ```
- AI 挂载使用统一的 `citizen.reloadJobAI()`
- 移除未使用的 `ModJobs` import

**效果**: `WAREHOUSE` 现在映射到 `"merchant"`，设施建成后自动分配完成。

## 3. 数据流（修复后）

### 职业变更链路（GUI 手动切换）

```
用户点击"建造师"
  → TownHallMenu.clickMenuButton(buttonId)
    → data.setJob("builder")          ✅ CitizenData
    → entity.syncFromData(data)       ✅ SynchedEntityData (新增)
    → citizen.reloadJobAI()            ✅ 原子替换 AI (新增)
      → removeGoal(oldGoal)
      → currentJobGoal = new BuilderWorkGoal(this)
      → addGoal(1, newGoal)
    → 玩家收到确认消息                 ✅
```

### 设施建成自动分配链路

```
ConstructionSite.complete()
  → assignNearbyUnemployed(buildingTypeId)
    → BuildingType.fromId("warehouse").getRequiredJobId()
      → "merchant"                    ✅ 不再是 null
    → data.setJob("merchant")         ✅
    → citizen.syncFromData(data)      ✅
    → citizen.reloadJobAI()           ✅
```

### 市民加入世界链路

```
EntityJoinLevelEvent
  → CitizenEvents.onEntityJoin()
    → citizen.reloadJobAI()           ✅ 统一入口
      → (失业) 什么也不做
      → (已分配) removeGoal + createGoal + addGoal
```
