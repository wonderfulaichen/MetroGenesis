## 1. 设计定位

- **玩家交互**：自由绘制矩形路径，贝塞尔曲线引擎生成平滑道路；道路提升物流效率（设计方案 §1.2/§6.2）。
- **俯视 RTS 视角**：按 M 进入市长俯视模式划区/画路（设计方案 §1.3）。
- **物流网络**：基于道路网格的批量运输，无实体配送员（设计方案 §6.2）。
- **当前真实状态**：手动画路是可玩核心路径；**自动连路是空桩**（见 §4）。

## 3. 架构 — 撤销系统（来自架构决策 §4）

### 当前方案（2026-06-11）
每段铺路操作独享 `ChangeStorage`，精确记录每个方块变更：
- `roadUndoStack` / `roadRedoStack`（`Stack<ChangeStorage>`）
- Ctrl+Z → `storage.undo(serverLevel, null)`；Ctrl+Y → `storage.redo(serverLevel)`

### 已知问题
1. **交互冲突**：`MayorBookScreen` 打开时 undo 后世界渲染不刷新
2. **漏块**：路基录制在实心层提前 `break`
3. **交叉路口干扰**：共享节点的路撤销后另一条接口断裂
4. **redo 局限**：不支持多段

### 理想方案（Phase 6+）
`ChangeStorage` 做录制，`RoadData` 做校验。撤销时先用 `ChangeStorage` 还原，再 `removeSegment` + `rebuildNode` 修复交叉点。

## 4. 已知缺口（🟡 / ❌）

> 术语纠正（功能完成度审计）：道路缺口是**方块自定义**，不是材质/贴图问题。

- ❌ **自动连路管线是空桩**：`road/handlers/PipelineRunner.java` 整文件为 `// TODO: implement actual road generation pipeline`，`runPipeline()` 是 no-op；`RoadPipelineController` 反复调用它但什么都不做（扫描结构 → A* 寻路 → 铺路 全未实现）。
- ❌ **方块未自定义（非材质）**：`ROAD_SURFACE = SMOOTH_STONE`、`ROAD_CURB = STONE_BRICKS`、`ROAD_BASE = DIRT` 硬编码在 `RoadBuilder` / `RoadTemplate` / `RoadPlacer` / `RoadTemplateManager` 四处，未使用 `init/` 注册的自定义方块，也无配置项。
- ⚠️ **模板无玩家入口**：`RoadTemplateManager.setActiveTemplate` 全项目仅管理器自己调用；玩家没有切换直道 / 十字 / 自定义样式的 UI。`getAutoTemplate`（3+ 连接自动选十字）是否真正接到 `rebuildNode` 存疑。
- ⚠️ `RoadBlueprintBuilder` 多处 `return null`；`RoadPlacer` 疑似只取模板 Z=0 层。

## 5. 未来规划（P1）

1. 填 `PipelineRunner` 自动连路（扫描结构 → A* 寻路 → 铺路）。
2. 方块自定义：在 `init/` 注册 road 专用方块，替换硬编码原版方块。
3. 给 `RoadTemplateManager` 加玩家切换入口（直道 / 十字 / 自定义）。
4. 无缝道路街面：与 Zone 生长的 Seamless 体系（`LEFT_WALL` / `RIGHT_WALL` / `CORNER`）对齐，形成连续街面。
