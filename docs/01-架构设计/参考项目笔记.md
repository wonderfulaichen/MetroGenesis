## 1. MineColonies

### 建筑放置管线

```
Blueprint (.blueprint NBT)
  → CreativeStructureHandler (instant placement)
    → StructurePlacer (iterates blueprint positions)
      → Manager.addToQueue(new PlaceStructureOperation(placer, null))
```

### 关键 API

| API | 用途 |
|-----|------|
| `BlueprintUtil.readBlueprintFromNBT()` | 从 NBT 读取蓝图 |
| `BlueprintUtil.writeBlueprintToNBT()` | 写入世界为蓝图 |
| `BlueprintUtil.createBlueprint()` | 从世界区域创建蓝图 |
| `StructurePacks.getBlueprint()` | 从数据包加载蓝图 |
| `CreativeStructureHandler` | 创造模式即时放置 |
| `PlacementSettings` | 旋转/镜像设置 |
| `PlaceStructureOperation` | 分 tick 放置操作 |
| `StructurePlacer` | 蓝图迭代执行器 |
| `BlueprintMapping` | 风格→路径映射 |

### 建筑蓝图文件

位置：`src/main/resources/blueprints/minecollections/<style>/<category>/<building><level>.blueprint`
- 24 种风格（acacia, medievaloak, nordic 等）
- 分类：agriculture, craftsmanship, education, fundamentals, military, mystic, infrastructure
- 等级：1~5 级（townhall5.blueprint 等）

## 2. Structurize

### 可复用基础设施

| 类 | 用途 |
|----|------|
| `RotationMirror` | 8 种旋转/镜像状态枚举 + applyToPos |
| `FacingFixer` | 方块朝向的镜像修复 |
| `ChangeStorage` | 方块前后状态快照 → 撤销/重做 |
| `UndoOperation` / `RedoOperation` | 撤销/重做操作封装 |
| `Manager.changeQueue` | 每个玩家的撤销缓存队列 |
| `Manager.addToQueue()` | 往 tick 操作池添加任务 |
| `PlaceStructureOperation` | 5 阶段分 tick 放置管线 |
| `Blueprint.setRotationMirror()` | 蓝图整体旋转/镜像变换 |
| `ItemScanAnalyzer` | 扫描分析工具 |

### 撤销系统原理

```
放置方块 → ChangeStorage 记录 pre/post state → Manager.addToUndoRedoCache
Ctrl+Z → UndoRedoMessage → Manager.undo() → UndoOperation.apply() → storage.undo()
→ 还原方块 → 产生 undo storage（用于 redo）→ 加入缓存
```

## 3. citybound（Plan-Based Undo）

citybound 不做方块录制，而是用 plan stack：
- 每个操作保存完整 project plan
- undo = pop last plan
- redo = push back
- 操作变更时**从当前所有活跃 plan 重新推导世界状态**

优势：
- 无方块录制 → 无漏块风险
- 多段交叉天然支持（每段独立 plan）
- 缺点：复杂度高，需全局 rebuild
