## 一、界面分析

### 1.1 当前界面结构

- **尺寸**: 248 x 196 像素
- **背景**: 纯代码绘制 `0xE01A1A2E` (深紫黑色)
- **边框**: 单线条 `0xFF4488FF` (蓝色)
- **标题**: "✦ 扫描结果" (金色 §6)

### 1.2 信息层级

1. **标题区** (y: +8): 金色标题
2. **尺寸信息** (y: +24): 尺寸和方块数
3. **方向信息** (y: +38): 入口方向、旋转、镜像状态
4. **预览区** (y: +50~130): 3D线框预览
5. **操作提示** (y: +132): 快捷键提示
6. **输入区** (y: +158~180): 名称输入框 + 按钮
7. **底部信息** (y: +186): 范围坐标

### 1.3 当前颜色方案

```
背景: #1A1A2E (RGB: 26, 26, 46) - 深紫黑
边框: #4488FF (RGB: 68, 136, 255) - 亮蓝
标题: §6 (金色)
信息: §7 (灰色) + §f (白色)
预览背景: #1A1A2E
线框: #884488FF (半透明蓝)
入口高亮: #88FFCC00 (半透明黄)
```

### 1.4 问题诊断

**视觉问题：**
- 背景过于简单，纯色填充缺乏层次感
- 边框单调，缺乏装饰性
- 按钮使用Unicode字符，缺乏图形化
- 线框渲染过于简单，缺乏立体感
- 部分文字颜色与背景对比度不足

**布局问题：**
- 标题区和信息区文字过多，略显拥挤
- 旋转/镜像按钮排列过密
- 预览区高度(80px)可能不足以展示3D效果
- 输入框和按钮在同一行，空间紧张

**交互问题：**
- 按钮缺乏悬停状态和点击反馈
- 缺少界面打开/关闭动画
- 操作提示文字过小且颜色过暗

## 二、设计规范

### 2.1 视觉目标

- 保持MetroGenesis的科技感主题
- 符合MC原版GUI风格习惯
- 提升界面专业度和美观度
- 改善用户体验和交互反馈

### 2.2 技术约束

- Minecraft Forge 1.20.1
- 纯Java渲染（无外部库）
- 性能优先（避免复杂渲染）
- 兼容不同分辨率

### 2.3 颜色系统

```java
// 背景系统
C_BG_DARK = 0xE012122A;      // 最深背景 #12122A
C_BG_PRIMARY = 0xE01A1A2E;   // 主背景 #1A1A2E
C_BG_SECONDARY = 0xD0222240; // 次背景 #222240
C_BG_LIGHT = 0xC02A2A48;     // 浅背景 #2A2A48

// 边框系统
C_BORDER = 0xFF3377EE;       // 主边框 #3377EE
C_BORDER_LIGHT = 0xFF4488FF; // 浅边框 #4488FF
C_BORDER_DARK = 0xFF2266CC;  // 深边框 #2266CC

// 强调色
C_ACCENT = 0xFFFFCC00;       // 金色强调 #FFCC00
C_ACCENT_LIGHT = 0xFFFFDD44; // 浅金色 #FFDD44
C_ACCENT_DARK = 0xFFDDAA00;  // 深金色 #DDAA00

// 状态色
C_SUCCESS = 0xFF44CC88;      // 成功绿 #44CC88
C_SUCCESS_HOVER = 0xFF55DDAA;
C_DANGER = 0xFFFF6644;       // 危险红 #FF6644
C_DANGER_HOVER = 0xFFFF7755;
C_WARNING = 0xFFFFAA22;      // 警告橙 #FFAA22

// 文字颜色
C_TEXT_PRIMARY = 0xFFFFFFFF;
C_TEXT_SECONDARY = 0xFFD0D0D0;
C_TEXT_TERTIARY = 0xFF808890;
C_TEXT_DISABLED = 0xFF555555;
```

### 2.5 交互规范

| 组件 | 状态 | 效果 |
|------|------|------|
| 按钮 | 普通 | C_BORDER 边框 |
| 按钮 | 悬停 | 颜色变亮 + 边框发光 |
| 按钮 | 点击 | 下沉 1px 效果 |
| 按钮 | 禁用 | C_TEXT_DISABLED 颜色 |
| 输入框 | 普通 | 深色背景 + 光标 |
| 输入框 | 聚焦 | 边框高亮 + 光标闪烁 |

## 三、纹理方案

- `container/inventory.png`（背包）

### 3.4 调色板


### 3.5 渐变方案

```

┌──────────────────────┐

│  白 → 浅灰           │
│  浅灰 → 中灰         │
│  中灰 → 深灰         │
│  深灰 (136,136,136)  │  ← C_BTN_INNER_BOTTOM

面板背景渐变（从上到下）:
┌──────────────────────┐
│  深 (18,18,42)       │  ← C_BG_DARK
│  深 → 主色           │
│  主色 (26,26,46)     │  ← C_BG_PRIMARY
└──────────────────────┘
```

## 四、实现优先级

| 优先级 | 内容 | 估计工时 |
|--------|------|---------|
| P0 | 颜色系统常量定义 + 基础背景/边框 | 0.5天 |
| P1 | 按钮纹理渲染（渐变+悬停+按下） | 1天 |
| P1 | 输入框美化（边框+光标+聚焦态） | 0.5天 |
| P2 | 预览区增强（立体感知） | 1.5天 |
| P2 | 滚动条/进度条 | 0.5天 |
| P2 | 过渡动画 | 2天 |
