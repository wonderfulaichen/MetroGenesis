## 1. 核心设计理念

### 1.1 玩家身份：市长，不是包工头

区别于 MineColonies 的"玩家手动放建筑、搬材料"模式，MetroGenesis 的玩家是**城市最高行政长官**：

- 不直接放置建筑 → 通过建设局划区，建筑自动生长 / 从目录选择放置
- 不直接分配工人 → 部门自动招人
- 不手动搬材料 → 物流网络自动配送
- 管理方式：看面板、定政策、调税率

### 1.2 核心特色

- **价值驱动经济**：每种物品/服务拥有内在城市价值（C-Value），市民收入与消费基于该价值，无通胀压力
- **完全自给自足的市民**：所有建筑材料、食物、工具均由市民生产，无需玩家手动挖矿砍树
- **政府部门自动管理**：财政部、建设局、商务部、人口局、规划局等NPC部门自动执行任务
- **物流网络代替配送员**：基于道路网格的批量运输系统，无实体卡顿
- **自由绘制区域 + 道路加速**：玩家自由绘制矩形功能区，道路提高物流效率
- **丰富的政策与事件系统**：玩家通过政策面板点选发展方向，动态事件带来挑战与机遇

### 1.3 控制方式：俯视角

参考《城市：天际线》的交互模式：

- 玩家在俯视角下划区或放置建筑
- 区内建筑根据需求和政策自动生成/升级
- 玩家查看全局数据面板，而非逐个点建筑
- 按 M 键切换至"市长俯视模式"（RTS 视角）

## 2. 五大部门系统

### 2.1 部门架构总览

部门系统采用统一模块化框架：

```java
interface Department<T extends DepartmentData> {
    String getId();
    T getData();
    void onDailyTick(CityState city);
    Screen getPanel();
    List<Policy> getPolicies();
}

abstract class DepartmentData {
    abstract void save(CompoundTag tag);
    abstract void load(CompoundTag tag);
}
```

```
市长（玩家）
├── 财政部   → 税收、国库、货币政策
├── 建设局   → 区域规划、自动建设
├── 商务部   → 供需看板、产业链、贸易
├── 人口局   → 移民、生育、人口统计
└── 规划局   → 科技树、政策研究
```

### 2.2 财政部（Treasury）

**职责**：管理城市财政

**玩家操作**：
- 调节各行业税率（个人所得税、企业所得税、消费税、地价税）
- 查看财政报告（收入/支出/结余占比）
- 政策工具：最低工资系数、社会补贴、发行国债
- 货币工具：央行基准利率调节

**数据**：国库余额(C-Value) / 每日收支流水 / 各行业税率配置

### 2.3 建设局（Construction Bureau）

**职责**：城市规划与建设

**玩家操作**：
- 划区：在区块网格上指定功能区（住宅/商业/工业/农业/公共）
- 自动填充：系统根据需求在区内放置基础建筑
- 基础设施：道路、路灯、管线规划
- 蓝图模式：保存/加载城市布局

**数据**：功能区类型 / 建筑密度和等级

### 2.4 商务部（Commerce Bureau）

**职责**：经济调节

**玩家操作**：
- 供需看板：每种商品的产量/消费量/库存/C-Value 趋势
- 价格干预：设定价格上限/下限
- 贸易路线：设置进出口协议

**数据**：商品注册表 / 每日供需统计 / 贸易协议列表

### 2.5 人口局（Population Bureau）

**职责**：人口管理

**玩家操作**：
- 人口统计：总人口、年龄分布、职业分布、失业率
- 移民政策：开启/关闭移民，设定吸引力预算
- 生育鼓励：发放生育奖金
- 移民筛选：按技能/教育程度过滤

### 2.6 规划局（Planning Bureau）

**职责**：科技与政策

**玩家操作**：
- 科技树：解锁新建筑/新政策/效率提升
- 政策研究：选择研究方向
- 城市法令：颁布临时政策

## 3. C-Value 经济引擎

### 3.1 核心公式

```
C-Value = 基准价值 × (1 + 科技系数) × 供需因子
```

- **基准价值**：每种物品管理员预设（面包=5，木头=2，铁锭=10）
- **科技系数**：由规划局科技提供
- **供需因子**：由商务部每日计算

```
供需因子 = 1.0 + (月消费量 - 月产量) / 月产量 × 弹性系数
```

- 供不应求 → 价格上浮（上限+20%）
- 供过于求 → 价格下跌（下限-20%）

### 3.2 市民收入

```
市民日收入 = 工作产出总价值 × 工作效率系数 × 城市工资基准
```

- 工作效率系数 = 0.5 + 技能等级 × 0.05（最高1.0）
- 工资基准由财政部设定（默认1.0，范围0.5~1.5）

### 3.3 政府税收

```
个人所得税 = ∑(市民日收入 × 个税率)      // 10%~30%
企业所得税 = ∑(企业利润 × 企税率)        // 5%~20%
消费税     = ∑(商品售价 × 消费税率)      // 5%~15%
地价税     = ∑(建筑每日 × 等级系数)
```

### 3.4 周期结算

```
每日凌晨结算（批量处理）：
① 所有市民工作产出 → 生成物品 + 个人收入
② 所有市民消费 → 消耗物品 + 扣除 C-Value
③ 更新供需统计
④ 计算新的供需因子
⑤ 更新所有物品的当日 C-Value
⑥ 各部门执行自动操作
⑦ 生育/移民/死亡检查
⑧ 保存数据到世界存档
```

## 5. 建筑系统

### 5.1 放置模式（Phase 0e）

参考 MineColonies 手动放置模式，去掉材料消耗：
- 玩家从管理书建筑分类目录选建筑
- 拖拽/点击放置到世界（带预览）
- 放置免费（不需要收集材料）
- 建筑模板用 MineColonies 的 `.blueprint` 文件
- 需求面板显示当前城市缺什么设施

### 5.2 放置管线

```
Blueprint (.blueprint NBT)
  → CreativeStructureHandler (instant placement)
    → StructurePlacer (iterates blueprint positions)
      → Manager.addToQueue(new PlaceStructureOperation(placer, null))
```

### 5.3 拼图式对齐建筑（Phase 3）

为了在 MC 方块世界中实现"连续街面"效果，引入拼图式对齐建筑体系。

**核心思路**：不是所有建筑都追求无缝，而是专门设计一批侧面平墙的建筑作为"拼图块"，在放置时自动对齐。

**SeamlessType 枚举**：
- `NONE` — 普通独立建筑（默认）
- `LEFT_WALL` — 左侧平墙，可贴左对齐
- `RIGHT_WALL` — 右侧平墙，可贴右对齐
- `BOTH_WALLS` — 双侧平墙，放在中间
- `INNER_CORNER` — L 型内转角
- `OUTER_CORNER` — L 型外转角

**风格包目录规范**：每个风格包增加 `seamless/` 子目录，存放拼图式建筑。

**放置吸附规则**：放置 LEFT_WALL 时检测右侧同行是否有建筑 → 自动侧面吸附。

**自动生长流程**：
```
划区 → 扫描可建地块
→ 沿道路先排 seamless 拼图块（形成连续街面）
→ 转角处自动选 CORNER 类建筑
→ 剩余空地用传统独立建筑填充
```

**玩家自定义**：关闭自动填充后，可从图鉴拖放任意建筑到区内指定位置。

## 5. 市民系统

### 5.1 市民属性

每个市民：姓名/性别/年龄(寿命1000~1500天)/职业/技能等级(1~10)/个人钱包(C-Value)/需求槽(饥饿/健康/快乐/知识)/工作建筑/住宅

### 5.2 市民行为逻辑

- 06:00 起床，吃早餐
- 07:00 前往工作建筑
- 07:30~17:00 工作
- 17:30 去商店/娱乐设施消费
- 20:00 回家，睡觉

### 5.3 生育与移民

- 自然生育：异性成年+空余住宅+幸福度>60%
- 移民：城市声望达标后每周有概率生成
- 寿命与死亡：达到上限死亡，可触发葬礼事件

## 6. 产业链与物流网络

### 6.1 产业链自动生成

放置工厂时系统自动检查上游原材料，若无则建设局自动规划。

### 6.2 物流网络（无实体配送员）

每日凌晨批量计算，使用网络流算法规划最优运输路径。
